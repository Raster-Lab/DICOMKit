> **Reading `⊘ not covered`:** a flag is output-tested only if a golden scenario exercises it. Each uncovered flag now states **why** — `network` (needs a live PACS/DICOMweb server), `non-deterministic` (fresh UIDs/timestamps, no stable golden), `no-write preview` (e.g. `--dry-run`), or `coverage gap` (offline-testable, not yet templated). Only the **coverage gap** flags can be driven down by contract-driven auto-generation; the network/non-deterministic ones are a permanent floor. For those, the per-tool **Verified App↔CLI parity** block is the authority — both adapters call the same DICOMKit engine, so the output matches *by construction* even without a golden.

> **Not-wired CLI tools** (e.g. `dicom-3d`, `dicom-measure`, `dicom-gateway`, `dicom-jpip`, `dicom-report`, `dicom-j2k`, `dicom-viewer`) have no DICOMStudio reimplementation, so output parity is undefined for them; they are out of scope here.
